# ProLine Aluminium Website

Marketing and lead-gen site for [ProLine Aluminium](https://prolinechch.co.nz), Christchurch-based aluminium joinery repair specialists.

Built with **Astro** + **Tailwind CSS v4**, deployed on **Cloudflare Pages**, with contact-form email delivery via **Resend**.

## Local development

```bash
npm install
npm run dev
```

Dev server runs at http://localhost:4321.

Notes:
- The contact form `POST /api/contact` is a **Cloudflare Pages Function** (see `functions/api/contact.ts`). It will 404 under `astro dev`. To test the full function locally:
  ```bash
  npm run build
  npx wrangler pages dev dist --compatibility-date=2024-12-01
  ```
  Then visit http://localhost:8788. Remember to set `RESEND_API_KEY` (see §Env vars below) before the form can actually send.

## Brand

- **Palette:** cool metallic greys (`ink-50` → `ink-950`) + chrome silver accents (`chrome-400` → `chrome-700`). Defined as CSS custom properties in `src/styles/global.css` and exposed as Tailwind utilities via `@theme`.
- **Type:** Inter Variable (body), Space Grotesk Variable (display).
- **Motion:** custom `ease-out-expo`, scroll-reveal via IntersectionObserver, respects `prefers-reduced-motion`.

## Deployment to Cloudflare Pages

### 1. Push to GitHub

Repo should contain this directory. See the git setup section at the bottom.

### 2. Connect in Cloudflare dashboard

1. **Cloudflare dashboard → Workers & Pages → Create → Pages → Connect to Git**.
2. Select the repo. If this directory is nested inside a larger repo, set the **root directory** to `Website`.
3. Framework preset: **Astro**. It will auto-fill:
   - Build command: `npm run build`
   - Build output directory: `dist`
4. **Save and deploy.**

Cloudflare will give you a URL like `proline-aluminium.pages.dev`. Use that as the temporary domain until `prolinechch.co.nz` is registered and pointed.

### 3. Env vars

In **Pages → Settings → Environment variables → Production**, set:

| Variable              | Required | Purpose                                                                                 |
| --------------------- | -------- | --------------------------------------------------------------------------------------- |
| `RESEND_API_KEY`      | Yes      | Resend API key (secret). Generate at https://resend.com/api-keys.                       |
| `CONTACT_FROM_EMAIL`  | No       | Sender. Format `"ProLine <website@prolinechch.co.nz>"`. Requires domain verification in Resend. Defaults to `onboarding@resend.dev` which works immediately but shows as resend.dev. |
| `CONTACT_TO_EMAIL`    | No       | Recipient. Defaults to `michael@prolinealuminium.co.nz`.                                |

Tick **"Encrypt"** for `RESEND_API_KEY`. Redeploy after adding.

### 4. Custom domain

Once `prolinechch.co.nz` DNS is in Cloudflare:

**Pages → Custom domains → Set up a custom domain → prolinechch.co.nz** (and `www.prolinechch.co.nz` if desired).

Cloudflare auto-creates the DNS records and certificate.

### 5. Resend domain verification (for branded sender)

To send emails from `@prolinechch.co.nz` rather than the generic `@resend.dev` sender:

1. **Resend dashboard → Domains → Add domain →** `prolinechch.co.nz`.
2. Add the TXT/MX/CNAME records they generate, into Cloudflare DNS.
3. Once verified, update `CONTACT_FROM_EMAIL` to `"ProLine <website@prolinechch.co.nz>"` and redeploy.

## Scripts

- `scripts/trace-logo.mjs`: regenerate `public/logo.svg` and `src/assets/logo.svg` from `../Logo.jpeg` (run if the client supplies a new logo).
- `scripts/generate-assets.mjs`: regenerate `favicon.svg`, `apple-touch-icon.png`, and `og-image.png` from the current logo and brand tokens.

```bash
node scripts/trace-logo.mjs
node scripts/generate-assets.mjs
```

## Accessibility

- Keyboard-accessible nav with visible focus rings.
- Semantic landmarks (`<header>`, `<main>`, `<footer>`, `aria-label`s).
- `prefers-reduced-motion` respected for transitions and scroll-reveal.
- Tap targets ≥ 44×44 px on mobile.
- Images have descriptive `alt` text.
